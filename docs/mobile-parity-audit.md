# Mobile Parity Audit

Objective: the mobile app must manage multiple hosts, agents, projects, threads, files, settings, and attention workflows with feature coverage comparable to the desktop app, while using mobile-first navigation and interaction patterns.

Status: implementation coverage is broad, but completion is not fully validated until the device and connected-host checks at the end of this audit pass.

## Success Criteria

- Multiple hosts can be paired, listed, inspected, reconnected, edited, made default, and removed.
- Projects can be listed across hosts, created, opened, edited, archived, deleted, and searched.
- Agent threads can be listed, filtered, opened, created, continued, archived, deleted, handed off, interrupted, and resumed after disconnects.
- Runtime attention workflows cover approvals, user-input requests, errors, notifications, and notification deep links.
- Project workspace workflows cover file browsing/editing, git status/actions, branches, worktrees, pull requests, GitHub issues, scripts, terminal, browser, and diffs.
- Settings cover local mobile preferences and remote host/provider/tooling preferences.
- Mobile UI uses platform-appropriate tabs, stacks, safe areas, reachable controls, visible loading/error/empty states, retry paths, and native permission recovery.
- Verification covers static checks, unit tests, bundle export, and real-device or simulator workflows against a live host.

## Remaining Validation Before Completion

- Pair a fresh mobile install to a real desktop host using QR and relay/manual flows.
- Run a connected-host smoke test for project create, thread create/send, approval response, browser open/retry, script run, commit/push/PR paths, and issue thread creation. The built-in diagnostics cover config, orchestration snapshots, settings, tooling, file list, temporary file write/read/delete, git status, and terminal open/resize/close.
- Validate iOS and Android notification permission prompts, foreground local notifications, background notification taps, and cold-start deep links.
- Validate physical-device camera capture and photo-library selection, including denied, ask-again, and blocked permission states.
- Perform side-by-side desktop/mobile parity review against the desktop surfaces in `apps/web/src/components`.
- Capture mobile screenshots or simulator recordings for the primary screens at small and large phone widths.
